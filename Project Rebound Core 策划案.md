# 一、游戏策划案

## 1. 游戏概述

- **名称：**《弹射核心》（Project: Rebound Core）
- **类型：** 像素风 / 物理弹射 / Roguelike / 刷宝
- **核心玩法：** “一发入魂”。玩家通过调整角度发射一枚初始子弹，利用子弹在场景内的无限反弹、分裂与特效触发，在规定时间内全清场景内的方块。
- **核心卖点：**
  - **极致解压：** 数百个方块被连续撞击碎裂的视觉与听觉反馈。
  - **刷宝乐趣：** 类似暗黑系列的词缀系统，追求“神装”搭配。
  - **构筑深度：** 技能与配件组合出的全屏特效（割草体验）。

------

## 2. 核心玩法循环 (Loop)

1. **整备阶段：** 玩家在基地安装配件、升级技能。
2. **发射阶段：** 观察方块布局，调整角度，发射子弹。
3. **自动战斗：** 子弹在场景内自动反弹，触发配件效果（分裂、爆炸、中毒）。
4. **实时干预：** 玩家在子弹运行期间释放主动技能（如加速、改变重力、手动引爆）。
5. **结算/收获：** 获取掉落配件，解锁新关卡。

------

## 3. 系统设计深度化

### 3.1 配件系统 (Gear System)

配件分为 **[核心]、[涂层]、[引擎]、[挂载]** 四个部位：

- **基础属性：** 攻击力（单次撞击伤害）、弹射速度、子弹体积、暴击率。
- **高级特效 (Odin 实现配置)：**
  - **分裂词缀：** 撞击N次后，分裂出X枚子弹。
  - **元素词缀：** 附带燃烧（持续伤害）、冰冻（减速方块或使其易碎）、闪电（跳跃连锁伤害）。
  - **穿透词缀：** 一定概率穿透方块不反弹。
  - 等等...

### 3.2 技能系统 (Skill System)

玩家拥有能量条，随时间回复：

- **超频 (Overdrive)：** 5秒内子弹速度翻倍。
- **燃烧区 (Fire Zone)：** 在当前子弹位置生成燃烧区域，对区域内的方块每秒造成X火焰伤害。
- **瞄准 (Aim)：** 将子弹前进方向改为距离最近的方块所在位置。
- 等等...

### 3.3 关卡设计 (Level Design)

- **动态环境：** 除了普通方块，加入**传送门**、**减速区**、**加速轨道**等等特殊区域。
- **方块类型：**
  - **石块：** 需要撞击多次。
  - **炸药桶：** 被撞击后对周围方块造成AOE伤害。
  - **宝箱怪：** 极高血量，击碎后必掉稀有配件。

------

## 4. 技术实现方案 (Unity 6000+)

- **物理引擎：** 使用 Physics2D.CircleCast 进行检测，并手动计算反射向量。
- **DoTween 应用：**
  - **方块碎裂：** 撞击时通过 DoTween 实现细微的“抖动”和“缩放”效果。
  - **UI 动效：** 掉落配件时的弹跳感、血条减少的平滑过渡。

- **Odin应用：**
  - **配件编辑器：** 利用 Odin 制作可视化配件编辑器，无需改代码即可快速配置多种词缀。
  - **关卡配置：** 使用 Odin 的 `TableList` 功能，快速编辑每一关的方块波次和掉落概率。

------

## 5. 表现力（Juice）设计

为了达到“解压”的效果，必须注重以下细节：

1. **屏幕震动 (Camera Shake)：** 强力撞击或爆炸时产生不同幅度的震动。
2. **停顿感 (Hit Stop)：** 子弹击碎高级方块时，游戏瞬间暂停 0.02 秒，增加打击感。
3. **动态音效：** 随着子弹速度变快，碰撞音调逐渐升高（类似钢琴爬音），形成情绪上的递进。
4. **像素粒子：** 使用 Unity VFX Graph 制作大量像素碎屑迸发。

# 二、游戏开发阶段

### 🧱 第一阶段：核心物理与 MVP (最小可行性产品)

- **目标：** 实现“发射 -> 弹射 -> 击碎方块”的基础循环。
- **核心任务：**
  - 搭建基础 2D 场景（墙壁、发射器、测试方块）。
  - 编写基于 `Physics2D.CircleCast` 的自定义子弹移动与反射逻辑（解决原生 Rigidbody2D 在高速下容易穿模的痛点）。
  - 实现最基础的碰撞检测、扣血与方块销毁。

### ⚙️ 第二阶段：数据架构与 Odin 工具链搭建

- **目标：** 建立底层数据框架，告别硬编码，实现“所见即所得”的配置。
- **核心任务：**
  - 利用 `ScriptableObject` 设计配件（Gear）和技能（Skill）的数据结构。
  - 引入 Odin Inspector，编写自定义可视化面板（Editor Window），让你能像搭积木一样配置出“撞击3次后分裂出2发附带燃烧效果的子弹”这种复杂词缀。
  - 搭建基础的事件总线（Event Center），解耦子弹触发和特效逻辑。

### ⚔️ 第三阶段：词缀、技能与机制实装

- **目标：** 注入游戏的“肉”，让战斗变得丰富多样。
- **核心任务：**
  - 实装各种配件词缀（分裂、爆炸、穿透、元素附着）。
  - 制作能量系统与主动技能（超频、瞄准、燃烧区）。
  - 开发关卡生成器，利用 Odin 的 `TableList` 配置波次，引入特殊方块（炸药桶、宝箱怪、加速/减速区）。

### 🧃 第四阶段：“Juice” 表现力注入 (核心爽点)

- **目标：** 落实你策划案中的“极致解压”，让游戏不仅好玩，而且“好爽”。
- **核心任务：**
  - **DoTween 动效：** 方块受击抖动、UI 弹跳反馈。
  - **手感调优：** 加入 Hit Stop（子弹击碎高级方块时的 0.02 秒顿帧）和 Camera Shake（屏幕震动）。
  - **视觉与听觉：** 使用 VFX Graph 制作像素碎片爆裂效果，编写音频管理器实现随着速度或连击数升调的动态音效。

### 🔄 第五阶段：UI、经济系统与完整循环

- **目标：** 将所有孤立的系统串联成一个完整的商业化游戏标准闭环。
- **核心任务：**
  - 制作主界面和“整备阶段”的基地UI（装备穿戴、词缀洗练）。
  - 实现局内掉落拾取与结算画面。
  - 关卡进度保存与存档系统建设。
